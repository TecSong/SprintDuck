# Hackathon Delivery Plan

## Deadline

交付截止时间：2026-05-31 10:00 CST。

当前时间盒约 16 小时。产品定位已经扩大到“隐私优先的求职工作台 Agent”，但 Demo 必须收敛到一个稳定、可信、能讲清楚价值的核心闭环。

## Scope Decision

Demo 不展示“全自动投递”或真实 Boss 直聘/脉脉连接器。

本次黑客松应展示：

> 用户把简历和目标 JD 放进本地工作台，选择隐私模式后，Agent 生成证据化求职诊断：岗位匹配、简历优化方向、面试冲刺计划、投递沟通话术，并把下一步动作放进求职 pipeline。

这个 Demo 能支撑大定位，但不承诺短时间内无法稳定交付的平台自动化。

## Demo Story

1. 用户打开 SprintDuckAgent，看到“隐私优先求职工作台”。
2. 用户选择隐私模式：纯本地、脱敏后调用外部模型、完整上下文调用外部模型。
3. 用户加载或粘贴一份中文简历和一份目标 JD。
4. Agent 显示将使用哪些数据、是否会外发、外发给哪个模型服务商。
5. Agent 生成求职诊断报告：
   - JD 匹配度和证据覆盖率。
   - 简历中已有证据和缺失证据。
   - 简历优化建议。
   - 1-7 天面试准备计划。
   - 招聘者开场白和跟进话术。
6. 用户把该岗位加入 pipeline，并看到下一步动作。
7. PPT 讲清楚后续路线：公司背调、Offer 谈判、平台连接器、自动投递都在 roadmap 中，但高风险动作必须人工确认。

## Priority Classification

### P0: Must Ship Before 2026-05-31 10:00

P0 的目标是保证现场 Demo 不翻车，并让评委理解产品已经从“面试教练”升级为“求职 Agent”。

| Task | Deliverable | Verification |
| --- | --- | --- |
| 产品文案重定位 | UI 从“面试冲刺教练”改为“隐私优先求职工作台” | 首页首屏和报告区域能表达新定位 |
| 隐私模式展示 | 增加隐私提示或模式选择：纯本地、脱敏外发、完整外发 | Demo 中能解释“隐私优先不是绝对不外发” |
| 演示样例数据 | 提供一键加载的中文简历 + JD 样例 | 现场无需手打长文本 |
| 复用现有面试报告 | 保持现有 readiness/gaps/sprint plan 稳定可用 | `make test` 通过，手动跑通一次 |
| 求职诊断包装 | 把报告解释为 JD 匹配、证据差距、准备计划 | 评委能看出这是求职链路的一部分 |
| 投递话术草稿 | 生成或展示招聘者开场白/跟进话术 | Demo 能覆盖“投递前准备” |
| PPT v1 | 8 页以内，围绕问题、方案、隐私、Demo、路线图 | 能在 5 分钟内讲完 |
| Dry run | 完整演示 2 次，记录 fallback 操作 | 演示脚本不依赖临场发挥 |

### P1: Ship If P0 Finishes Early

P1 用来增强“求职工作台”的观感，但不应影响 P0 稳定性。

| Task | Deliverable | Verification |
| --- | --- | --- |
| Pipeline 轻量看板 | 静态或本地状态的岗位卡片：已收藏、已投递、面试中 | 可以把 Demo 岗位加入 pipeline |
| 外部调用审计视觉 | 展示一次模型调用的目的、数据范围、目标服务 | 不要求完整审计系统 |
| 公司背调占位 | 基于用户输入的公司名生成背调清单 | 不依赖真实联网搜索 |
| Markdown 导出优化 | 导出文件名和内容改为求职诊断报告 | 下载后内容可读 |
| PPT 美化和截图 | 用本地 Demo 截图替换纯文字页 | contact sheet 观感统一 |

### P2: Explicitly Do Not Build For This Deadline

这些能力属于大产品方向，但不适合黑客松短时间实现。

- Boss 直聘、脉脉真实登录和自动化操作。
- 自动批量投递。
- 真实招聘者消息发送。
- 云端账号、同步、支付、团队协作。
- 完整本地加密数据库。
- PDF/DOCX 高质量解析和导出。
- 薪酬数据库和 Offer 市场价查询。
- 长期记忆、复杂向量检索、跨平台去重。

- 本地启动 Demo，完整走一遍样例。
- 记录 fallback：

  - 如果新增 UI 有问题，回退到现有 chat/report 核心流。
  - 如果投递话术来不及动态生成，使用模板卡片展示。

### Phase 4: PPT

Time box: 2026-05-30 23:30-2026-05-31 01:00 CST。

制作 8 页以内 PPT：

1. 标题：SprintDuckAgent，隐私优先的求职工作台 Agent。
2. 痛点：求职链路碎片化 + 求职数据高度敏感。
3. 核心洞察：隐私优先不是不联网，而是用户可控的数据外发。
4. 方案：本地工作台 + Agent skills + 可授权连接器。
5. Demo 流程：简历/JD -> 隐私模式 -> 诊断报告 -> 投递话术 -> 面试计划。
6. 技术架构：FastAPI + React + LLM providers + tools/skills/MCP boundary。
7. Roadmap：P0 本地闭环，P1 背调/pipeline，P2 平台连接器和人工确认投递。
8. 总结：让求职者在一个可信工作台里完成从岗位发现到 Offer 的关键决策。

## Demo Script

5 分钟版本：

1. 30 秒：说明问题。求职者的简历、JD、招聘者沟通和面试准备都分散，且数据敏感。
2. 45 秒：说明定位。SprintDuckAgent 是隐私优先求职工作台，默认本地保存，外部调用必须授权。
3. 2 分钟：演示核心流。加载样例简历/JD，展示证据化匹配、gap、冲刺计划。
4. 60 秒：演示投递话术和 pipeline 下一步，说明这连接了“准备”和“行动”。
5. 45 秒：讲路线图。平台连接器和自动投递会做，但必须 human-in-the-loop，不做黑盒群发。

## PPT Claim Spine

- 求职不是单点任务，而是一条高隐私、高决策成本的链路。
- 传统 AI 改简历工具只解决片段问题，无法沉淀候选人上下文。
- SprintDuckAgent 把简历、JD、证据、面试准备和投递动作放到一个本地工作台。
- 隐私优先的核心是“外发可见、可控、可拒绝”。
- 黑客松 Demo 证明最小闭环：从 JD 到求职诊断和下一步行动。
- 后续扩展通过 tools、skills、MCP 和平台连接器完成，但所有高影响动作都需要用户确认。

## Kill Criteria

遇到以下情况立即降级，不继续扩大范围：

- 任一 P0 变更导致现有报告无法生成。
- 模型 API 不稳定超过 15 分钟。
- UI 新增模块影响主流程输入或报告展示。
- PPT 超过 8 页仍无法在 5 分钟讲完。
- 平台连接器需要真实账号、验证码或规避反自动化机制。

## Final Deliverables

- 可运行 Demo：本地 Web 工作台。
- 样例输入：中文简历 + 中文 JD。
- Demo 脚本：5 分钟版本。
- PPT：8 页以内。
- Git commit：包含 P0 Demo 代码和 PPT/文档产物。
- 验证结果：`make test`，以及一次手动 Demo 走查记录。
